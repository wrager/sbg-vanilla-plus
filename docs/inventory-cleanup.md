# Автоочистка инвентаря

Модуль `inventoryCleanup` автоматически удаляет лишние предметы из инвентаря перед изучением точки.

## Для игрока

### Когда срабатывает

Очистка запускается **перед каждым нажатием кнопки «Изучить»** (Discover), если кнопка доступна (не disabled). Если кнопка неактивна (точка вне зоны действия, кулдаун) — очистка **не** срабатывает. При простановке (Deploy) очистка также **не** срабатывает.

### Условие запуска

Очистка происходит, только если в инвентаре **мало свободных слотов**. Порог задаётся в настройках параметром «Мин. свободных слотов» (по умолчанию 100).

Пример: лимит инвентаря 3000, в инвентаре 2950 предметов, порог — 100. Свободных слотов: 50, что меньше 100 — очистка сработает.

### Что удаляется

Удаляются только **лишние** предметы сверх установленных лимитов:

| Тип предмет  | Как задаётся лимит         | Значение -1     |
| ------------ | -------------------------- | --------------- |
| Ядра (Cores) | Отдельный лимит на уровень | Без ограничений |
| Катализаторы | Отдельный лимит на уровень | Без ограничений |

**Ключи и мётлы не удаляются** — они всегда защищены от автоочистки. Удаление ключей будет доступно после реализации модуля «Избранные точки», который позволит защитить ключи к конкретным точкам.

По умолчанию все лимиты установлены в -1 (без ограничений), так что без настройки ничего не удаляется.

### Порядок удаления (FIFO)

Когда количество предметов превышает лимит, удаляются предметы **с начала инвентаря** — те, что были получены раньше.

Пример: лимит ядер 5-го уровня = 10. В инвентаре два стека:

```
Стек 1: 8 ядер 5 ур. (получен раньше)
Стек 2: 7 ядер 5 ур. (получен позже)
Всего: 15, лимит: 10, лишних: 5
→ Удалить из стека 1: 5 шт.
```

Для ключей порядок будет аналогичным, когда удаление ключей будет включено.

### Уведомления

После очистки в верхней части экрана появляется тост-уведомление на 3 секунды:

```
Очистка (13): Я5 ×10, К3 ×3   (русский)
Очистка (13): Co5 ×10, Ca3 ×3 (английский)
```

- Число в скобках — общее количество удалённых предметов
- `Я`/`Co` — ядра, `К`/`Ca` — катализаторы
- Цифра после буквы — уровень предмета
- `×10` — количество удалённых предметов этого типа

### Настройки

Настройки доступны через кнопку «Настроить» рядом с модулем на экране настроек SVP. Экран настроек позволяет задать:

- **Мин. свободных слотов** — при каком количестве свободных слотов запускать очистку
- **Ядра** — лимит для каждого уровня (1–10)
- **Катализаторы** — лимит для каждого уровня (1–10)

Настройки сохраняются в `localStorage` и применяются мгновенно.

### Планы на будущее

- **Избранные точки** — модуль для защиты ключей к определённым точкам от удаления (аналог CUI). После его реализации станет доступно удаление ключей с лимитом.
- **Разделение ключей по принадлежности** — отдельные лимиты для ключей союзных и вражеских точек. Требует запроса `/api/point` для определения команды точки, поэтому будет работать по кнопке (ручная очистка), а не автоматически перед discover.

## Техническая реализация

### Источники данных

- **Количество предметов и лимит инвентаря** — читаются из DOM (`#self-info__inv`, `#self-info__inv-lim`)
- **Содержимое инвентаря** — читается из `localStorage['inventory-cache']`, JSON-массив объектов:

  ```
  { g: guid, t: type, l: level|pointGuid, a: amount }
  ```

  - `t=1` — ядра (`l` = числовой уровень)
  - `t=2` — катализаторы (`l` = числовой уровень)
  - `t=3` — ключи (`l` = строковый GUID точки)
  - `t=4` — мётлы (игнорируются)

### Алгоритм

1. Прочитать `currentCount` и `inventoryLimit` из DOM
2. Если `inventoryLimit - currentCount >= minFreeSlots` — выйти (места достаточно)
3. Загрузить `inventory-cache` из localStorage, распарсить
4. Для ядер и катализаторов: сгруппировать по уровню, для каждого уровня определить излишек относительно лимита, пометить на удаление (FIFO)
5. Ключи и мётлы — пропускаются (удаление отключено)
6. Показать тост с итогом удаления
