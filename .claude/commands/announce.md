Составить короткий анонс изменений с версии $ARGUMENTS (например: `/announce 0.2.0`).

## Процесс

1. Определить все теги от указанной версии до HEAD: `git tag --sort=version:refname`.
2. Собрать коммиты `git log v<версия>..HEAD --oneline --no-merges`.
3. Сформировать текст анонса.

## Правила анонса

Формат - простой текст без маркированных списков, каждый пункт с новой строки через пустую строку.

Новые модули описывать как: Добавлен функционал "название" - краткое описание что делает (1 предложение).

Багфиксы одного модуля агрегировать в один пункт: Багфиксы модуля "название".

Удалять всё неинтересное пользователям: детали реализации, внутренние рефакторинги, изменения правил/CI, переименования категорий.

Не раздувать тривиальные изменения (исправление флага, опечатки) - либо в агрегированные багфиксы, либо убрать.

Отдельный коммит не означает отдельный пункт - группировать по фичам/темам.

Описывать итоговый результат для пользователя, не процесс разработки.

Не делить по версиям - один плоский текст.

Только символы с клавиатуры: кавычки ", дефис - как разделитель. Без длинного тире, без спецсимволов, без маркеров списка.

Язык: русский.
