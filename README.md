Юзерскрипт для улучшения стандартного интерфейса [SBG](https://sbg-game.ru). Минимальная инвазия — не перекраивает UI, а дополняет ванильную игру тем, чего в ней не хватает.

Модульная архитектура: каждая фича изолирована и переключается независимо. Не нужна фича или не нравится — просто выключите. Если после обновления игры что-то сломалось — отключите один модуль, остальные продолжат работать.

## Установка

Доступны два способа:

- **[SBG Scout](https://github.com/wrager/sbg-scout/releases/latest)** — Android-приложение: игра в WebView + встроенный менеджер юзерскриптов, Vanilla+ ставится и обновляется автоматически.
- **Менеджер юзерскриптов в браузере:**
  1. Поставить менеджер юзерскриптов: [Tampermonkey](https://www.tampermonkey.net/), [Violentmonkey](https://violentmonkey.github.io/), [Userscripts](https://apps.apple.com/app/id1463298887)
  2. Открыть [последний релиз](https://github.com/wrager/sbg-vanilla-plus/releases/latest) и установить `.user.js`

## Модули

### Интерфейс

| Фича                         | ID                      | Описание                                                                                                                                 |
| ---------------------------- | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Улучшенный главный экран     | enhancedMainScreen      | Компактная верхняя панель: ник под кнопками, инвентарь в ОРПЦ, шестерёнка вместо «Настройки», кнопка атаки по центру                     |
| Улучшенный UI попапа точки   | enhancedPointPopupUi    | Крупные кнопки, мелкий текст, авто-простановка убрана от случайных нажатий                                                               |
| Улучшенный UI вкладки ключей | enhancedRefsTab         | Карточка ключа разносится на 3 фиксированные строки — высота не зависит от длины имени владельца                                         |
| Группировка тостов ошибок    | groupErrorToasts        | Группирует одинаковые тосты ошибок в один со счётчиком вместо накопления                                                                 |
| Убрать «Закрыть» в атаке     | removeAttackCloseButton | Убирает кнопку «Закрыть» в режиме атаки, чтобы не нажать её случайно вместо «Огонь!». Выход из режима — повторный клик по кнопке «Атака» |

| Фича                      | ID                  | Описание                                                                                                                              |

| Автоочистка инвентаря     | inventoryCleanup    | Автоматически удаляет лишние предметы при изучении точек. При включенном модуле «Избранные точки» доступна возможность очистки ключей |
| Экран не гаснет           | keepScreenOn        | Экран не гаснет во время игры (Wake Lock API)                                                                                         |

| Ключи на карте            | refsOnMap           | Просмотр и управление точками с ключами на карте на любом масштабе                                                                    |

## Настройки

Настройки игры → «Настройки SBG Vanilla+» → «Открыть» → полноэкранная панель с toggle для каждой фичи. Переключение в реальном времени.

## Обратная связь

Нашли баг или хотите предложить фичу?

- **Баг-репорт из игры (рекомендуется):** откройте настройки скрипта (⚙) → нажмите «Сообщить об ошибке». Скрипт автоматически скопирует диагностику (версия, браузер, модули, лог ошибок) в буфер обмена и откроет форму на GitHub — останется только описать проблему и вставить диагностику из буфера (Ctrl+V).
- [Создать баг-репорт вручную](https://github.com/wrager/sbg-vanilla-plus/issues/new?template=bug_report.yml)
- [Предложить фичу](https://github.com/wrager/sbg-vanilla-plus/issues/new?template=feature_request.yml)

## Разработка

```bash
npm install
npm run dev        # Dev server
npm run build      # Сборка скрипта в dist/
npm run typecheck  # Проверка типов
npm run lint       # ESLint
npm run test       # Jest
```

### Dev-сервер + менеджер юзерскриптов

1. Запустить `npm run dev`
2. vite-plugin-monkey откроет страницу с прокси-скриптом — установить его в менеджере юзерскриптов
3. Открыть `sbg-game.ru/app/` — скрипт загружается с localhost
4. При изменении кода dev-сервер пересобирает — обновить страницу игры для применения

Dev-скрипт имеет пометку `[DEV]` в названии. Продакшн `downloadURL`/`updateURL` не включаются в dev-режиме.

## Добавление фичи

1. Создать `src/modules/<name>/<name>.ts`, реализующий `IFeatureModule`
2. Если нужен CSS — создать `src/modules/<name>/styles.css`
3. Импортировать модуль в `src/entry.ts`

## Отладка

Добавить `#svp-disabled=1` к URL страницы игры — скрипт полностью отключится (удобно для сравнения поведения с оригинальным интерфейсом). Состояние сохраняется в `sessionStorage` на время сессии. Для повторного включения — `#svp-disabled=0`.

Пример: `https://sbg-game.ru/app/#svp-disabled=1`

## Совместимость

Скрипт совместим с SBG v0.6.0 и v0.6.1. На v0.6.1 модули, чьи функции игра реализовала сама, автоматически отключаются и собираются в отдельную группу настроек с указанием причины — настройки сохранятся, если игра откатится на v0.6.0. На более новых версиях скрипт спрашивает подтверждение перед запуском. При обновлении игры сломавшиеся модули не блокируют остальные — каждый модуль изолирован через try/catch.
