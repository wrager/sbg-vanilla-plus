# Ключи на карте (refsOnMap)

Просмотр и управление ключами в инвентаре через визуализацию точек на карте на любом масштабе.

## Для игрока

### Запуск

Кнопка «На карте» в инвентаре, на вкладке ключей. Кнопка вставляется рядом с нативной `#inventory-sort`.

### Что делает

- Открывает full-screen viewer поверх игровой карты: подгружает все ключи из `inventory-cache`, для каждой стопки рисует кружок в координатах точки. Один кружок = одна стопка ключей (несколько стопок одной точки рисуются отдельно).
- Кружок раскрашен цветом команды точки. Команды грузятся двумя путями параллельно: (a) active pull через `/api/point` для видимых в текущем extent ref-точек при открытии viewer и на каждый `moveend` карты; (b) enrichment через перехват игрового `/api/inview` — когда игра дёргает этот эндпоинт сама, hook ловит ответ и сразу пишет команды в кэш, минуя `/api/point`. Worker делает double-check `teamCache.has` перед каждым `/api/point` — если /inview успел положить команду между извлечением guid'а из очереди и реальным запросом, лишний fetch не отправляется.
- Клик по точке выбирает её. При клике на перекрывающиеся точки (несколько кружков под одним пикселем) ВЫБИРАЮТСЯ ВСЕ невыбранные; уже выбранные не трогаются. Повторный клик НЕ снимает выделение — для очистки выбора есть кнопка «Отменить» под «Корзиной».
- Справа внизу появляется кнопка «Корзина» с числами `X (X ключей)` — точек и ключей, готовых к удалению. Справа над «Корзиной» — блок-сводка по выбору и блок radio-режима «своих». Клик по корзине удаляет deletable-стопки через DELETE `/api/inventory`. После успешного удаления выделение со всех оставшихся (защищённых) точек сбрасывается.
- Закрытие — кнопка `[x]` снизу.

### Маркеры на кружке

В центре кружка отображается ОДИН элемент в порядке приоритета:

1. **Иконка замочка** (alpha 70%) — точка с lock-флагом в инвентаре (бит 1 поля `f`).
2. **Иконка звезды** (alpha 70%) — точка с favorite-флагом (бит 0 поля `f`).
3. **Обе иконки наложены, каждая alpha 50%** — точка одновременно locked И favorited.
4. **Текст «=1»** — выделенная точка, у которой режим «Оставлять 1 ключ для своих» после удаления оставит ровно 1 ключ.
5. **Число `amount`** — текущее количество ключей в стопке. Виден только при сильном приближении (zoom >= 15).
6. Пусто — при низком зуме и без маркеров выше.

Иконки lock/star видны и для выделенных, и для невыделенных точек — они отражают inventory-state, а не selection. Текст «=1» — только для выделенных.

### Цвета кружка

- Команда 1/2/3 — соответствующий цвет из `--team-N` (палитра игры).
- Команда ещё не загружена (`team === undefined`) — `--team-0` из палитры игры (`#444` на светлой теме, `#CCC` на тёмной). Совпадает по тону с нативным слоем points под нашим: пока мы не знаем команду, точка не выделяется на фоне нативной отрисовки.
- Нейтральная точка (`team === null`) — явный `#666666`. Известное состояние, не зависит от темы: отличимо от unloaded в обеих темах.

Невыделенный кружок: fill teamColor с alpha 25%, stroke teamColor solid.

Выделенный кружок, который реально пойдёт в удаление: fill `#BB7100` с alpha 50%, stroke `#BB7100` solid, радиус +40%. Заливка и обводка разной насыщенности — пользователь видит две зоны, а не сплошной оранжевый блок.

Для выделенных, у которых после удаления НЕ уйдёт ни одного ключа (locked/own/unknown-защищённые либо keepOne-полностью_защищённые), fill полностью прозрачный — только обводка выделения держит индикацию selected, оранжевый под заливкой не намекает на «к удалению».

### Сводка по выбору

Блок справа внизу, виден когда выбрана хотя бы одна стопка. До шести строк (lock/own/unknown/keepone — опциональные, видны только когда соответствующий bucket не пуст):

- **«Выделено: X (X ключей). Из них:»** — суммарный выбор без учёта защит. Точки считаются по уникальным `pointGuid`.
- **«X (X ключей) защищено»** — точки с lock.
- **«X (X ключей) красные/зелёные/синие и незащищённые, но не удалятся»** — точки команды игрока (цветное прилагательное под `getPlayerTeam()`). Опциональная, видна ТОЛЬКО при режиме «Не удалять своих» или «Оставлять 1 ключ для своих» И когда own-bucket не пуст. Уточнение «незащищённые, но не удалятся» отражает: эти точки своей команды НЕ являются locked (locked-свои уже учтены в строке «защищено»), но не пойдут в payload из-за самого режима.
- **«X (X ключей) неизвестного цвета»** — выбранные точки, для которых `team` не удалось загрузить. Опциональная, видна ТОЛЬКО при protective-режиме.
- **«X последних ключей останутся»** — сумма ключей, которые останутся в инвентаре благодаря правилу «Оставлять 1 ключ для своих». Видна ТОЛЬКО при режиме `keepOne` и когда правило сработало.
- **«К удалению: X (Y ключей)»** — итог в payload DELETE: X уникальных точек, Y ключей суммарно. Видна всегда при наличии выбора (в том числе при 0 — когда всё защищено).

Это же число дублируется на кнопке «Корзина» снизу, но в сводке оно стоит рядом с разбивкой по защитам — пользователь видит «откуда X получилось» в одном месте.

При защите всего выбора (lock/own/unknown/keepOne) кнопка покажет `0 (0 ключей)`, клик по ней покажет тост-разъяснение и не уйдёт в DELETE.

После успешного DELETE выделение со ВСЕХ оставшихся в source фич сбрасывается — защищённые точки остаются видны, но не выделены.

### Режим защиты своих (radio)

Блок из трёх взаимоисключающих опций справа в viewer, виден когда есть выбор. Сверху вниз (от меньшей защиты к большей):

1. **«Удалять красные/зелёные/синие»** (`delete`) — без защит. Все выделенные ключи удаляются.
2. **«Оставлять 1 ключ для красных/зелёных/синих»** (`keepOne`, дефолт) — у каждой точки своей команды остаётся минимум 1 ключ. Точки своей с одним ключом не трогаются; с N>1 ключами обрезаются до 1. Не-свои удаляются полностью.
3. **«Не удалять красные/зелёные/синие»** (`keep`) — точки своей команды полностью защищены. Не-свои удаляются полностью.

Цвет в названии динамический — берётся через `getPlayerTeam()` (`red`/`green`/`blue`). При `playerTeam=null` — fallback «свои» / «own team».

**State персистентный:** `localStorage['svp_refsOnMap'].ownTeamMode`. Дефолт `keepOne`. Миграция со старого формата (`keepOwnTeam`/`keepOneKey`) не делается — записи в старом формате трактуются как невалидные, отдаётся дефолт.

### Защита от удаления

- **Замочек (нативный SBG 0.6.1)** — ключи locked-точек игнорируются при удалении даже если выбраны. lock-семантика per-point: одна locked-стопка защищает ВСЕ стопки точки. Действует независимо от режима radio.
- **Режим radio** — определяет, что делать с точками своей команды (см. выше).
- **Тост по итогу удаления.** Полный успех — «Y ключей от X точек успешно удалены». Ошибка запроса — «⚠️ Ошибка удаления Y ключей от X точек». Сервер возвращает либо 200 OK на весь запрос, либо ошибку — частичного успеха не бывает, поэтому смешанный двухстрочный текст не нужен. Перечисление защит (Locked / Зелёные / Не загружен цвет / Оставлять 1 ключ) в тост не выводится — эта информация уже видна в selection-info на момент клика по trash.

Инвариант, проверяемый отдельной серией critical-safety тестов:

- guid из protected-bucket (lock/own/unknown) НЕ может попасть в payload DELETE ни при одной комбинации.
- При режиме `keepOne` сумма payload по СВОЕЙ точке СТРОГО МЕНЬШЕ суммарного inventory amount. То есть после DELETE у каждой выделенной своей точки в инвентаре остаётся минимум 1 ключ.

### Прогресс-бар и блокировки

Active pull через `/api/point` worker батчами по 5 (delay 100 мс между батчами) запускается при `showViewer` и на каждом `moveend`. `/inview`-перехват параллельно ловит игровые ответы.

Пока в очереди что-то есть:

- Видим прогресс-бар сверху по центру (счётчик `done / total`).
- **Кнопка «Корзина» disabled — ТОЛЬКО при режиме `keep`/`keepOne` И когда хотя бы одна выбранная точка реально в очереди загрузки команды.** Фоновая загрузка команд для невыбранных точек кнопку не блокирует.
- **Клик по карте работает всегда.** При protective-режиме выбранная точка с team=undefined попадёт в unknown-bucket (fail-safe).
- Pan и zoom карты работают как обычно.

После выработки очереди прогресс-бар скрывается.

### Алгоритм keepOne (classifyFeatures)

Единый классификатор `classifyFeatures` применяется ко всем выделенным фичам и возвращает per-feature результат с полями `isLocked` / `isFavorited` / `deletion` / `toDelete` / `toSurvive`. Этот результат — единственный источник правды для payload, текста селекшен-инфо, тоста и стилей карты (иконки/=1/alpha).

Логика классификации для выделенной фичи (в порядке проверки):

1. lock в инвентаре → `lockedProtected`.
2. mode = `keep`/`keepOne`, team = undefined → `unknownProtected` (fail-safe).
3. mode = `keep`, точка своей команды → `ownProtected`.
4. mode = `keepOne`, точка своей команды → keepOne-обрезка по группе:
   - `selectedAmount = sum amount стопок этой точки в группе`.
   - `inventoryTotal = sum amount всех стопок точки в инвентаре`.
   - `unselectedAmount = inventoryTotal - selectedAmount`.
   - `unselectedAmount >= 1` — защита уже выполнена, стопки → `fullyDeletable`.
   - `selectedAmount <= 1` — нечего удалять, всё → `keepOneTrimmed` с toDelete=0, toSurvive=amount.
   - Иначе distribute `selectedAmount - 1` ключей по убыванию amount (тай-брейк по guid). Стопки полностью удалённые → `fullyDeletable`, частично → `keepOneTrimmed` с toDelete>0, toSurvive=остаток.
5. Иначе (чужие, нейтральные) → `fullyDeletable`.

Невыделенные фичи всегда классифицируются как `nothingToDelete` (но `isLocked`/`isFavorited` читаются всегда — для иконок).

### Поведение при `playerTeam = null`

При protective-режиме (`keep`/`keepOne`) и `playerTeam=null` `handleDeleteClick` блокирует удаление с тостом «Не удалось определить команду игрока. Удаление заблокировано (переключите режим на „Удалять“ ...)». Своя команда не определяется, выполнить заявленную защиту нельзя, удалять без защиты — нарушение явного намерения пользователя.

### Значения `feature.team` и связь с bucket'ами

- `number` — конкретная команда (1..4). При protective-mode сравнивается с `playerTeam`: совпадение → own-bucket (keep) или keepOne-обрезка (keepOne), иначе → fullyDeletable.
- `null` — нейтральная точка (`data.te: null`). НЕ считается своей даже в protective-режиме — удаляется как чужая. В стилях рисуется явным `#666666`.
- `undefined` — команда не загружена (active pull не дошёл / fetch упал). При protective-режиме → `unknownProtected` (fail-safe). При delete — удаляется. В стилях рисуется через `--team-0` из палитры игры.

## Связанные модули

- `inventoryCleanup` — общая семантика lock-защиты. Та же функция `buildLockedPointGuids` из `core/inventoryCache.ts` используется для определения защищённых точек.
- `core/inventoryCache.buildFavoritedPointGuids` — аналогично для иконки звезды на карте.
- `core/playerTeam` — `getPlayerTeam()` для определения своей команды.
- `core/refsHighlightSync` — после успешного удаления ключей вызывается `syncRefsCountForPoints` для актуализации подписи на основной карте.
- `refsLayerSync` — использует тот же паттерн monkey-patch `window.fetch` (для `/api/discover`); refsOnMap перехватывает `/api/inview`, цепочка hook'ов не конфликтует.
