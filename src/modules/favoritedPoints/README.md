# Избранные точки (favoritedPoints)

Модуль позволяет отмечать игровые точки как избранные и защищает ключи от этих точек от удаления автоочисткой.

## Возможности для игрока

- **Звезда на попапе точки** — кнопка в попапе точки для добавления/удаления из избранного
- **Звёзды на ключах в инвентаре** — визуальная метка избранных ключей
- **Фильтр «Только избранные»** — чекбокс во вкладке ключей инвентаря. При активном фильтре показываются также избранные точки без ключей (данные подгружаются через `/api/point`)
- **Экспорт/импорт** — настройки модуля позволяют экспортировать список избранных в JSON и импортировать обратно (для миграции между устройствами)

> Защита ключей от избранных точек при рисовании линий переехала в отдельный модуль «Ограничения рисования» (`drawingRestrictions`).

## Хранилище

Совместимо с CUI: та же IndexedDB база `CUI`, тот же objectStore `favorites` (keyPath=`guid`). Запись — `{ guid, cooldown }`. Поле `cooldown` используется только CUI; наш модуль сохраняет его при upsert, чтобы не затереть данные CUI.

### In-memory кэш

При `init()` все записи загружаются из IDB в `memoryGuids: Set<string>`. Синхронные проверки (`isFavorited`, `getFavoritedGuids`) читают из кэша. Обновления (`addFavorite`, `removeFavorite`) записывают в IDB и обновляют кэш после коммита транзакции (`tx.oncomplete`).

### Count seal (защита от потери IDB)

При каждом изменении в localStorage записывается количество избранных (`svp_favorites_seal`). При загрузке если IDB пуста, а seal > 0 — значит данные потеряны (Android IDB wipe). В этом случае `snapshotLoaded = false` (блокирует удаление ключей автоочисткой) и показывается alert.

### Событие `svp:favorites-changed`

Диспатчится на `document` при каждом изменении. Используется для синхронизации UI (звёзды, фильтр, счётчик).

## 4-уровневая защита от удаления ключей

1. **isModuleActive** — ключи удаляются только если модуль включён И готов (status=ready)
2. **5-condition guard в calculateDeletions** — все 5 условий должны быть true: referencesEnabled, mode=fast, limit≠-1, snapshotReady, size>0
3. **Per-key filtering** — ключи от избранных точек исключаются из расчёта лимита ДО подсчёта
4. **Final API guard** — перед HTTP DELETE перечитывается свежий snapshot избранных и каждый ключ проверяется повторно

## Файловая структура

| Файл                 | Назначение                                                 |
| -------------------- | ---------------------------------------------------------- |
| `favoritedPoints.ts` | Определение модуля: init (загрузка из IDB), enable/disable |
| `starButton.ts`      | Звезда в попапе точки, MutationObserver на атрибуты попапа |
| `inventoryFilter.ts` | Фильтр и звёзды в инвентаре, MutationObserver на DOM       |
| `settingsUi.ts`      | UI настроек: счётчик и экспорт/импорт JSON                 |
| `debugHooks.ts`      | Opt-in отладочный API (window.svpFavs)                     |
| `styles.css`         | Стили звезды, фильтра, settingsUi                          |

## Взаимодействие с inventoryCleanup

`inventoryCleanup` использует `getFavoritedGuids()` и `isFavoritesSnapshotReady()` из `favoritesStore` (core) для определения какие ключи защищены. Прямого импорта между модулями нет — связь через core.
