# Улучшенный UI вкладки ключей (enhancedRefsTab)

Делает карточку ключа во вкладке References инвентаря визуально предсказуемой и блокирует кнопку зарядки, когда заряжать уже нечего.

## Возможности для игрока

- **3-строчная карточка.** Содержимое каждой карточки разносится на три фиксированные строки независимо от длины имени владельца. Высота карточки одинаковая для всех ключей — список не «прыгает» по вертикали.
- **Неактивная кнопка зарядки при полном заряде.** Если у точки заряд 100%, кнопка repair в карточке становится `disabled`. SBG 0.6.1 в нативе сделала кнопку «всегда активной» (refs/game-beta/script.js:3573 — условие сужено до проверки команды), и пользователь стучался в сервер впустую, чтобы получить отказ. Этот фикс возвращает прежнее предсказуемое поведение клиента.

## Архитектура

### Слой 1: 3-line layout (CSS)

`styles.css` — flexbox-разметка `.inventory__item-left` с `flex-direction: column` и фиксированной высотой строк. Никакого JS.

### Слой 2: repair-button override (JS)

`repairButtonOverride.ts` подписывается на DOM игры через scoped `MutationObserver` и блокирует кнопку, если энергия точки ≥ 100%.

**Scoped observer.** Наблюдатель вешается на `.inventory__content`, а не на `document.body`: иначе он будет триггериться на каждое изменение DOM игры, тогда как нам нужны только события вокруг карточек реф-вкладки.

**Опции наблюдения.** `{ subtree, childList, attributes, attributeFilter: ['disabled', 'class'] }`. `class` нужен, чтобы реагировать на появление класса `loaded` на карточке (refs/game-beta/script.js:3568 — игра ставит его после заполнения `.iid-energy`); `disabled` — чтобы возвращать неактивное состояние, если игра сняла его в обработчике клика.

**Парсер процента.** `parseEnergyPercent` извлекает число из текста `.iid-energy` формата «`<percent>% @ <cores>`» (refs/game-beta/script.js:3550). Поддерживает дробную часть с точкой и запятой; невалидную строку (включая «загрузка...») возвращает как `null`.

### Защита от рекурсии

`WeakSet<HTMLButtonElement> selfDispatched` помечает кнопки, на которых наш обработчик только что выставил `disabled`. Микротаск (Promise resolve) очищает Set, чтобы наш собственный mutation не интерпретировался как нативное снятие и не вызывал повторный setAttribute в бесконечном цикле.

### Защита от race-conditions

`installGeneration` инкрементируется при каждом install/uninstall. `waitForElement('.inventory__content').then(...)` сравнивает свой generation с актуальным: если между ожиданием и резолвом случился disable, install пропускается. Без этого быстрый цикл enable→disable→enable оставлял бы «висящий» observer от первого enable.

## Файловая структура

| Файл                           | Назначение                                                        |
| ------------------------------ | ----------------------------------------------------------------- |
| `enhancedRefsTab.ts`           | Определение модуля: enable/disable, инжект/съём CSS, install hook |
| `repairButtonOverride.ts`      | MutationObserver, парсер `.iid-energy`, scoped DOM-overrides      |
| `repairButtonOverride.test.ts` | Тесты парсера и observer-сценариев                                |
| `styles.css`                   | 3-line flex-разметка карточки                                     |

## Настройки

Модуль не имеет настроек. Включён по умолчанию.
