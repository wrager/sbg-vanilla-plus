# Улучшенный UI попапа точки (enhancedPointPopupUi)

Удобный попап точки: крупные кнопки, мелкий текст, скрытая авто-простановка, кнопки избранного и блокировки.

## Возможности для игрока

- **Крупные игровые кнопки.** Нативные кнопки в `.i-buttons` (deploy / repair / draw / discover) растянуты до `min-height: 72px` для удобного попадания пальцем на мобильном.
- **Уменьшенный текст метаданных.** Строки в `.i-stat__entry` (расстояние, владелец, линии, регионы) сжаты до `0.7rem` - крупный текст для каждой строки занимает много места и оттесняет деплой-слайдер вниз. Строка `.i-stat__cores` исключена: ей нужен полный размер для читаемости уровней ядер.
- **Скрыта авто-простановка.** Кнопка `#magic-deploy-btn` уезжает в фиксированный левый нижний угол `32x32` - она не нужна большинству пользователей и провоцирует случайные тапы на деплой ядра автоматически без возможности подкрутить уровень.
- **Кнопки избранного и блокировки в попапе точки.** Звезда (favorite) и замок (locked) в правом нижнем углу `.i-image-box` рядом со счётчиком ключей. Клик переключает соответствующий нативный флаг SBG для всех стопок ключей точки в инвентаре через `POST /api/marks`.

## Кнопки fav/lock

Нативный API SBG 0.6.1 (`POST /api/marks`) помечает не точку, а конкретную стопку ключей в инвентаре (`item.g`). У одной точки может быть несколько стопок (после массовых сборов) с разными значениями битов в поле `f`.

### Поведение

- **Состояние читается агрегированно по всем стопкам точки.** Кнопка считается включённой, только если ВСЕ стопки имеют соответствующий бит. Совпадает с правилом `lockSupportAvailable` в `inventoryCleanup`, где mix-стопки трактуются как неполный lock.
- **Toggle применяется ко всем стопкам, чей текущий бит отличается от целевого.** Запросы идут sequential с задержкой `MARKS_RATE_LIMIT_MS = 1500` мс между ними (общая константа `core/marksApi`, обходит серверный rate-limit на marks). На время batch обе кнопки disabled.
- **Если у точки нет ключей в инвентаре - кнопки disabled.** Tooltip пустой; нативный API работает на уровне стопки, помечать нечего. После сбора первого ключа кнопки активируются автоматически (state перечитывается при следующем открытии попапа).
- **После batch состояние перечитывается из локально обновлённого `inventory-cache`.** `postMark` синхронизирует кэш по реальному ответу сервера: если сервер вернул `result: false` (toggle снял флаг противоположно ожиданию), визуальное состояние следует серверу.
- **Иконки нативные SBG.** Звезда `fa-star` / `fas-star` и замок `fas-lock-open` / `fas-lock` - те же FA-spritesheet'ы, что у нативного `.inventory__ref-actions` popover в инвентаре.

## Архитектура

### Lifecycle

`enable()` инжектит `styles.css` и вызывает `installPointMarkButtons()`. Установщик ждёт `.info.popup` через `waitForElement`, ставит `MutationObserver` на атрибуты `class` (hidden / visible) и `data-guid` (смена точки) и вставляет контейнер с двумя кнопками сразу после `#i-ref` в `.i-image-box`.

`disable()` снимает observer, аборт-сигналит click-listener'ы и удаляет контейнер.

### Reentrancy

`installGeneration` инкрементируется на каждом install/uninstall. Если `waitForElement.then()` приходит после disable - generation не совпал, обработчик skip'ает работу. `batchInProgress` блокирует повторные клики и не даёт `MutationObserver` пересоздавать DOM во время отправки запросов.

### Без подписки на внешние события

Состояние читается на изменении атрибутов попапа (открытие, смена точки). Нативный popover `inventory__ref-actions` живёт в инвентаре, не в попапе точки - параллельных изменений в открытом попапе не происходит. При закрытии и повторном открытии попапа state перечитывается из текущего `inventory-cache`.

## Файловая структура

| Файл                           | Назначение                                                                                                             |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| `enhancedPointPopupUi.ts`      | Определение модуля + lifecycle install/uninstall                                                                       |
| `pointMarkButtons.ts`          | DOM-инжект кнопок fav/lock, MutationObserver, агрегация state по стопкам, sequential POST /api/marks                   |
| `pointMarkButtons.test.ts`     | Тесты на install/uninstall, агрегацию state, click-toggle с mix-стопками, batch-throttle, обработку network/toggle off |
| `enhancedPointPopupUi.test.ts` | Тесты на инжект стилей и изоляцию своих селекторов от нативных `.inventory__ref-actions` / `.inventory__manage-amount` |
| `styles.css`                   | Стили крупных игровых кнопок, мелкого текста метаданных, фиксированной авто-простановки и контейнера кнопок fav/lock   |

## Настройки

Модуль не имеет настроек. Включён по умолчанию.
