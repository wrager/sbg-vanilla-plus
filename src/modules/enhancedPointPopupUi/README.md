# Улучшенный UI попапа точки (enhancedPointPopupUi)

Удобный попап точки: крупные кнопки, мелкий текст, скрытая авто-простановка, кнопки избранного и блокировки.

## Возможности для игрока

- **Крупные игровые кнопки.** Нативные кнопки в `.i-buttons` (deploy / repair / draw / discover) растянуты до `min-height: 72px` для удобного попадания пальцем на мобильном.
- **Уменьшенный текст метаданных.** Строки в `.i-stat__entry` (расстояние, владелец, линии, регионы) сжаты до `0.7rem` - крупный текст для каждой строки занимает много места и оттесняет деплой-слайдер вниз. Строка `.i-stat__cores` исключена: ей нужен полный размер для читаемости уровней ядер.
- **Скрыта авто-простановка.** Кнопка `#magic-deploy-btn` уезжает в фиксированный левый нижний угол `32x32` - она не нужна большинству пользователей и провоцирует случайные тапы на деплой ядра автоматически без возможности подкрутить уровень.
- **Звезда избранного в попапе точки (восстановление UX).** В прошлых версиях SVP/CUI избранное жило в локальном сторе скрипта и не передавалось серверу. Здесь та же UX-удобность звезды прямо в попапе точки, но теперь сразу записывается через нативный `POST /api/marks` SBG 0.6.1: пометка сохраняется на сервере и видна в нативном инвентаре игры.
- **Замок блокировки в попапе точки (НОВАЯ функция, не было в прошлых версиях).** Запирает все стопки ключей точки нативным замочком SBG. Locked-ключи защищены от удаления автоочисткой (`inventoryCleanup`) и не показываются в массовом удалении нативного `.inventory__manage-amount`. **Внимание:** случайное нажатие на уже filled-замок снимет защиту со ВСЕХ стопок этой точки, и при следующем срабатывании автоочистки ключи могут быть удалены без подтверждения. Подтверждающего диалога нет - cost-benefit для разовых ошибок: ручной re-tap проще, чем confirm на каждый toggle. Tap target расширен до 44x44 с gap 8px, чтобы свести к минимуму промахи.

## Кнопки fav/lock

Нативный API SBG 0.6.1 (`POST /api/marks`) помечает не точку, а конкретную стопку ключей в инвентаре (`item.g`). У одной точки может быть несколько стопок (после массовых сборов) с разными значениями битов в поле `f`.

### Поведение

- **Состояние читается агрегированно по всем стопкам точки.** Кнопка считается включённой, только если ВСЕ стопки имеют соответствующий бит. Совпадает с правилом `lockSupportAvailable` в `inventoryCleanup`, где mix-стопки трактуются как неполный lock.
- **Toggle применяется ко всем стопкам, чей текущий бит отличается от целевого.** Запросы идут sequential с задержкой `MARKS_RATE_LIMIT_MS = 1500` мс между ними (общая константа `core/marksApi`, обходит серверный rate-limit на marks). На время batch обе кнопки disabled.
- **Если у точки нет ключей в инвентаре - кнопки disabled.** Tooltip пустой; нативный API работает на уровне стопки, помечать нечего. После сбора первого ключа кнопки активируются автоматически (state перечитывается при следующем открытии попапа).
- **После batch состояние перечитывается из локально обновлённого `inventory-cache`.** `postMark` синхронизирует кэш по реальному ответу сервера: если сервер вернул `result: false` (toggle снял флаг противоположно ожиданию), визуальное состояние следует серверу.
- **Иконки нативные SBG.** Звезда `fa-star` / `fas-star` и замок `fas-lock-open` / `fas-lock` - те же FA-spritesheet'ы, что у нативного `.inventory__ref-actions` popover в инвентаре.

## Архитектура

### Lifecycle

`enable()` инжектит `styles.css` и вызывает `installPointMarkButtons()`. Установщик ждёт `.info.popup` через `waitForElement`, ставит `MutationObserver` на атрибуты `class` (hidden / visible) и `data-guid` (смена точки) и вставляет контейнер с двумя кнопками сразу после `#i-ref` в `.i-image-box`.

`disable()` снимает observer, аборт-сигналит click-listener'ы и удаляет контейнер.

### Reentrancy

`installGeneration` инкрементируется на каждом install/uninstall. Если `waitForElement.then()` приходит после disable - generation не совпал, обработчик skip'ает работу. `installAbortController` через `AbortSignal` сразу освобождает pending `waitForElement` при uninstall, не оставляя MutationObserver на documentElement до timeout. `batchInProgress` (Set<pointGuid>) блокирует повторные клики на той же точке и не даёт `MutationObserver` пересоздавать DOM во время отправки запросов; uninstall очищает Set и завершает висящий цикл через сверку generation на каждой awaited step.

### Подписка на inventory-cache

Состояние читается:

- На изменении атрибутов попапа (`class` hidden/visible, `data-guid` смена точки) - `MutationObserver` на самом попапе.
- На изменении `localStorage['inventory-cache']` - idempotent wrapper на `localStorage.setItem`, ставится один раз на жизнь страницы. Нативный `.inventory__ref-actions` popover в SBG 0.6.1 клонируется внутрь `.info.popup` (см. тест `enhancedPointPopupUi.test.ts`), и его toggle favorite/locked пишет в `inventory-cache` синхронно с открытым попапом - без подписки наш UI оставался бы stale до закрытия и повторного открытия. Та же подписка ловит discover (новая стопка - агрегат меняется) и срабатывание `inventoryCleanup`.

Wrapper не снимается на disable: соседи в цепочке `setItem` (`inventoryCleanup` тоже оборачивает) не пережили бы восстановление прототипа. `observedPopup === null` после uninstall - wrapper по факту no-op.

## Совместимость с CUI / EUI

CUI (`refs/cui/index.js:4078-4123`) инжектит `button.sbgcui_point_star` в тот же `.i-image-box`. При одновременно включённых SVP+CUI пользователь видит две похожих звезды:

- **`button.sbgcui_point_star`** - локальный favorite CUI. Состояние хранится в `localStorage['favorites']` (объект CUI), сервер о нём не знает. Используется CUI-фичами: список избранных в CUI-меню, фильтр в просмотре карты.
- **`.svp-point-mark-button[data-flag="favorite"]`** - нативный favorite SBG 0.6.1 (наш). Состояние - бит `f & 0b01` стопки в `inventory-cache`, идёт на сервер через `POST /api/marks`, видно в нативном инвентаре игры и в нативных списках.

Звёзды независимы: клик по нашей не меняет CUI-состояние и наоборот. Иконки разные (FA-spritesheet против FA-class у CUI) и наша звезда занимает правый верхний угол `.i-image-box`, CUI-звезда обычно появляется в другом месте, но при изменении CUI-вёрстки возможно частичное наложение - в этом случае правый верхний угол всегда наш.

EUI с favorites не работает, конфликта нет. SBG Scout - тот же скрипт-менеджер, что и в браузере, поведение не отличается.
