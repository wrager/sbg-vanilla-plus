# Ограничения рисования (drawingRestrictions)

Модуль скрывает нежелательные цели из списка рисования линий. Фильтрация делается на клиенте: сам запрос к серверу не модифицируется — перехватывается только ответ `GET /api/draw`.

## Возможности для игрока

- **Защита избранных точек** — три режима:
  - _Без защиты_ — избранные ключи в списке рисования показываются наравне с остальными.
  - _Защищать только последний ключ_ — из списка убирается только избранная цель, где у игрока остался ровно 1 ключ (чтобы не потратить последний при рисовании). В этом режиме показывается toast-уведомление, когда что-то скрыто.
  - _Скрывать все избранные цели_ — из списка убираются любые избранные цели, независимо от количества ключей.
- **Ограничение по расстоянию** — числовое поле в метрах (0 = без лимита). Цели дальше порога скрываются из списка.
- **Режим «звезда»** — одна точка фиксируется как центр звезды. Когда игрок стоит у центра, список рисования не фильтруется (все линии из центра — часть звезды). Когда игрок стоит у другой точки, в списке остаётся только центр звезды — игрок не может случайно нарисовать «не звёздную» линию. Центр подсвечивается жёлтым кольцом на карте. Управление через две кнопки в попапе точки:
  - основная кнопка «назначить/снять центром» видна всегда
  - дополнительная кнопка «сбросить центр» видна только в попапе не-центральной точки, когда центр уже назначен — позволяет снять центр, не переназначая текущую точку

## Архитектура

- `drawFilter.ts` — единственный перехватчик `GET /api/draw`. На `enable()` оборачивает `window.fetch`, на `disable()` восстанавливает оригинал. Двойная установка идемпотентна.
- `filterRules.ts` — чистые предикаты (без DOM/fetch). `buildPredicates({ settings, favorites })` возвращает активные правила, `applyPredicates()` — композиция AND. Правила, для которых настройка не активна, не добавляются в массив — `drawFilter` при пустом массиве отдаёт оригинальный Response без парсинга JSON.
- Toast о скрытии последнего ключа триггерится ровно тогда, когда сработало правило `protectLastKey`; массовые режимы (`hideAllFavorites`, `maxDistanceMeters`) тосты не показывают — скрытия там ожидаемые.

## Настройки

- `favProtectionMode: 'off' | 'protectLastKey' | 'hideAllFavorites'` — режим защиты избранных (default: `protectLastKey`)
- `maxDistanceMeters: number` — максимальная дистанция в метрах, 0 отключает фильтр (default: `0`)
- localStorage key: `svp_drawingRestrictions`

### Миграция из favoritedPoints

При первом запуске новый модуль читает `svp_favoritedPoints.hideLastFavRef` и переносит значение:

- `true` → `favProtectionMode: 'protectLastKey'` (прежнее поведение сохраняется)
- `false` → `favProtectionMode: 'off'`

После сохранения `svp_drawingRestrictions` миграция больше не срабатывает.

## Файловая структура

| Файл                     | Назначение                                                                      |
| ------------------------ | ------------------------------------------------------------------------------- |
| `drawingRestrictions.ts` | Определение модуля: init/enable/disable                                         |
| `settings.ts`            | Загрузка/сохранение настроек + миграция `hideLastFavRef`                        |
| `settingsUi.ts`          | UI настроек: radio с тремя режимами, числовое поле макс. расстояния             |
| `filterRules.ts`         | Чистые предикаты фильтрации `IDrawEntry` и функция композиции                   |
| `drawFilter.ts`          | Обёртка `window.fetch` для `GET /api/draw`, применяет предикаты, toast          |
| `starCenter.ts`          | Хранилище GUID центра звезды в localStorage + событие `svp:star-center-changed` |
| `starCenterButton.ts`    | Две контекстные кнопки в попапе точки (назначить/снять/сбросить центр)          |
| `starCenterHighlight.ts` | Жёлтое кольцо вокруг иконки центра звезды на OL-карте (слой `points`)           |
| `styles.css`             | Стили кнопки «Настроить», кнопок звезды и модалки настроек                      |
