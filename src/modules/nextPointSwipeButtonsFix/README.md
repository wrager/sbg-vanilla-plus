# nextPointSwipeButtonsFix

Восстанавливает срабатывание кнопок попапа точки после переключения на следующую точку.

## Симптом

После открытия попапа другой точки (через свайп - нативный игровой или наш `improvedNextPointSwipe`, или через любой код, вызывающий `showInfo`) первый тап на любой кнопке попапа (`#draw`, `#discover`, `#repair`, `#i-navigate`, `#deploy` и др.) иногда не срабатывает: кнопка визуально нажимается (видны pointerdown/pointerup), но `click` event не возникает, и обработчик игры не вызывается. Повторный тап через 1-2 секунды работает нормально.

## Корневая причина

WebView и Android Chrome probabilistically не синтезируют `click` event из touch sequence на кнопке, если она находится под элементом, претерпевшим массивный DOM mutation burst в недавнем прошлом. Игровой `showInfo` (refs/game/script.js:2084) при переключении точки делает 300+ DOM-изменений: `splide.refresh()` пересоздаёт деплой-карусель, обновляются все тексты попапа, меняются inline-стили, происходят layout shifts. Браузер в течение примерно 1.5-2 секунд после такого burst иногда пропускает `click` synthesis из последующих touch sequences.

Pattern не deterministic: на одних и тех же действиях сегодня click срабатывает, завтра нет. Точное условие подавления из JS не наблюдаемо. Симптом одинаковый и для нативного свайпа игры, и для нашей кнопки/свайпа - все они идут через `showInfo`.

## Защита

Polyfill `core/clickSynthesis.ts`. На pointerup проверяем через 80мс, fired ли `click`. Не fired - диспатчим синтетический `MouseEvent('click')` с теми же координатами вручную. Игровой обработчик (например, `$('#draw').on('click', ...)`) обрабатывает синтетический click идентично нативно-синтезированному.

Защита от двойного срабатывания: per-pointerup временный listener в capture phase ловит реальный click. Если пришёл - флаг ставится, ручной dispatch блокируется. Не пришёл - диспатчим. Listener удаляется через 80мс, не накапливается.

## Покрываемые кнопки

ВСЕ `<button>` элементы внутри `.info.popup`. На enable модуль ставит fallback на каждую существующую кнопку. Дальше MutationObserver на subtree попапа добавляет fallback на новые кнопки, появляющиеся при пересоздании частей попапа в `showInfo` (cores list, deploy slider slides и пр.).
